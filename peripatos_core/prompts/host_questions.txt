# Core Mission
Read the section overview to understand the paper's structure, then plan a multi-chapter podcast interview. Group the paper's content into 3-7 logical, thematic chapters. Each chapter should cover a distinct topic or phase of the paper.

Chapter titles must be descriptive and content-based (e.g., "The Architecture of GPT-4" not "Chapter 2"). Titles must be 80 characters or fewer.

For each chapter, write 2-5 interview questions that probe that chapter's topic. Order questions logically within each chapter. Cover different aspects such as methodology, results, and implications. Ensure each question is answerable from the paper's content.

# Output Format
Output strictly as valid JSON:
{"chapters": [
  {"title": "Chapter title here", "questions": ["Question 1", "Question 2"]}
]}

# Constraints
- Do not use generic chapter titles like "Chapter 1" or "Introduction" — each title must be descriptive of the content.
- Each chapter must have between 2 and 5 questions.
- Questions must be answerable from the paper content — do not ask speculative or impossible questions.
- The output must be valid JSON with no additional text before or after.
